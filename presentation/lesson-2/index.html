What the heck is HTML anyway?

  • Hyper-Text Markup Language

- You might remember HTTP - HyperText Transfer Protocol, ie a method of transferring stuff like HTML.

- What does markup mean? It's a term used in other industries and usually means something different every time, but it's always a way of describing something. Architecture has something called ADML (Arch Desc Mark Lang). It just means a way of explaining to a computer how something is structured.

- HTML is just text, with some funny symbols, that fit together in a structure to describe a page, and it's content, to a browser.

HTML is based on XML

What the heck is XML?!

  • eXtensible Markup Language

<parent-tag>
	<child-tag attribute="value">Content!</child-tag>
</parent-tag>

- XML allows for anything following this structure

- A pair of "elements" or "tags" (with chevrons), one opening, one closing

- A tag can have an attribute with a value

- A tag can have content

- Tags can be nested inside each other infinitely, and a parent can have multiple children

Terminology:

- Nesting/nested - when one tag lives inside another, it is "nested"

- Often referred to as Parent and Child. Also Grandparent, or Grandchild sometimes. Anything more than that is Ancestor or Descendant.

Realistic example of XML


<apartment floor="14">
	<apartment-name>Grand Manor</apartment-name>

	<rooms floor="upstairs">
		<bathrooms>1</bathrooms>
		<bedrooms>2</bedrooms>
		<kitchens>0</kitchens>
	</rooms>

	<rooms floor="downstairs">
		<bathrooms>1</bathrooms>
		<bedrooms>0</bedrooms>
		<kitchens>1</kitchens>
	</rooms>
</apartment>

- It's representing some data of some kind

- Just take a minute to look at this, and see if it makes sense

- There's a few different ways you could represent this same data, eg you could but the name in an attribute, like "floor"

- XML doesn't define anything about what you call your tags, or how you nest them. It doesn't care. It's up to me how I structure it AND ALSO up to me how I interpret it...

- HTML is just XML that a browser can understand.

HTML is just a "defined" XML


<h1>My article<h1>

<article>
	<section class="large-section">My article content</section>
</article>

- What does "defined" mean? It just means it has some rules to it. The types of tags are interpretted in a certain way - those tags MEAN something to the browser reading it. [NEXT FRAGMENT]

- There is a certain structure to it, in what tags can go inside other tags. What attributes can go on tags.

- HTML5 made things a lot easier. The rules are way less strict now. You can pretty much use any tag name.

- Browser are super forgiving these days. That means if you make mistakes in your tags, it will try to fix them.

- But that's no excuse, keep your markup clean and tidy and easy to read and you won't have awful errors to debug. Don't be sloppy, I'm going to be strict about that because it's an excellent habit to have.

Basic page set up

  • We have to set up a "document"
  • The document is built up from "markup"
  • It contains a "doctype", a "head" and a "body"
  • The "head" tells your browser some more information, such as language, title, favicon
  • The "body" contains all of the content for your websites. Images, text, etc
  • This is the sort of thing you tend to set up once, and never look at again

- This is definitely one of those "be aware, don't remember" things.

- I'm going to take you through this, but we're not going to look at it again for a little while.

- I have intentionally left out explaining DOCTYPE because it is the least important thing to know about. Easier to see an example.


<!DOCTYPE html>
<html lang="en">
<head>
	<meta charset="UTF-8">
	<title>Document</title>
</head>
<body>
	<!-- content -->
</body>
</html>

- Head is instructions for your web browser

- Body is what will be displayed

How about CSS?

What is it?

  • Cascading Style Sheets
  • Allows you to "style" your markup (HTML)

With CSS you can...

  • Add colours to elements and content
  • Add backgrounds colours and images to elements
  • Change the layout and positioning of elements
  • Change the size of elements and content
  • Add animations and transitions
  • Add minor interactivity such as hover styles

- CSS is becoming constantly more powerful (and therefore somewhat complex)

- In a basic sense, it allows you to control the look at feel of your web page

- (Understood? Does everyone understand it's role? Can I explain more? Maybe Codepen?)

Basic CSS


h1 { color: red; }

article { width: 300px; }

section { background-color: grey; }

.my-class-name { ... }

#my-id-name { ... }

#my-parent-id .my-child-class { ... }

p.specific-element { ... }

- We'll look at some CSS but I'm not going to explain it all too well. If you understand it great, but if not, don't worry – we'll be coming back to this next time and going into a lot more detail

- The first part is a "selector" which tells the browser what part of the HTML you are talking about

- The part inside the curly brackets is the "styling" – what you want to do with that selector

- There is multiple ways of "selecting" things, with classes and ids and more

- Try not to get too hung up on this. We're going to do more on this next time. I just wanted to show you it.
